# 项目概述

## 一、项目简介

本项目是基于 **STM32F407ZGT6** 微控制器的 **BMS（Battery Management System，电池管理系统）** 控制方案。项目使用 **STM32Cube HAL 库** 进行硬件抽象层驱动开发，搭载 **FreeRTOS** 实时操作系统实现多任务调度，适用于电动汽车或储能系统的电池管理与热管理控制。

### 硬件平台

| 项目         | 描述                               |
| ------------ | ---------------------------------- |
| **主控芯片** | STM32F407ZGT6 (Cortex-M4F, 168MHz) |
| **开发环境** | Keil MDK (ARMCC)                   |
| **HAL库**    | STM32F4xx HAL Driver               |
| **操作系统** | FreeRTOS (heap_4)                  |
| **调试接口** | USART1 (串口打印, printf 重定向)   |

### 系统时钟配置

| 时钟域           | 频率    | 说明                |
| ---------------- | ------- | ------------------- |
| **SYSCLK**       | 168 MHz | 系统主时钟，PLL输出 |
| **HCLK (AHB)**   | 168 MHz | AHB总线时钟，不分频 |
| **PCLK1 (APB1)** | 42 MHz  | APB1外设时钟，4分频 |
| **PCLK2 (APB2)** | 84 MHz  | APB2外设时钟，2分频 |
| **HSE**          | 8 MHz   | 外部高速晶振        |

> 时钟路径：HSE(8MHz) → PLL(M=4, N=168, P=2) → SYSCLK=168MHz
>
> 注：`stm32f4xx_hal_conf.h` 中 `HSE_VALUE` 默认为 25MHz，实际硬件晶振为 8MHz。PLL 参数验证：8MHz / 4 × 168 / 2 = 168MHz ✓

## 二、外设硬件连接描述

> 以下内容根据代码中已配置的外设定义自动生成，未参考原理图。

### 1. USART1 串口通信

| 功能      | 引脚     | 复用功能     | 方向 | 说明                 |
| --------- | -------- | ------------ | ---- | -------------------- |
| USART1_TX | **PA9**  | AF7 (USART1) | 输出 | 串口发送（调试打印） |
| USART1_RX | **PA10** | AF7 (USART1) | 输入 | 串口接收（指令输入） |

- **波特率**: 115200 bps（通过 `usart_init(baudrate)` 参数可调）
- **数据位**: 8位
- **停止位**: 1位
- **校验位**: 无
- **硬件流控**: 无
- **中断优先级**: 抢占3，子优先级3
- **接收缓冲**: 200 字节环形缓冲 + `\r\n` 行结束符检测
- **用途**: 调试信息输出（`printf` 重定向）与指令接收

### 2. CAN1 总线通信

| 功能    | 引脚     | 复用功能   | 方向 | 说明      |
| ------- | -------- | ---------- | ---- | --------- |
| CAN1_TX | **PA12** | AF9 (CAN1) | 输出 | CAN1 发送 |
| CAN1_RX | **PA11** | AF9 (CAN1) | 输入 | CAN1 接收 |

- **波特率**: 500 Kbps
- **分频系数 (Prescaler)**: 6
- **时间段1 (BS1)**: 9 TQ
- **时间段2 (BS2)**: 4 TQ
- **同步跳转宽度 (SJW)**: 1 TQ
- **波特率验证**: 42MHz / 6 / (9+4+1) = 500Kbps ✓
- **工作模式**: 正常模式 (NORMAL)
- **AutoBusOff**: 使能（自动离线恢复）
- **AutoRetransmission**: 使能（自动重传）
- **过滤器组**: ID 掩码模式, 32位, 接收所有帧 (ID=0, Mask=0), FIFO0
- **过滤器范围**: CAN1 使用 0~13 号（SlaveStartFilterBank=14）
- **用途**: BMS 与电池采集芯片、VCU 等外部设备的 CAN 通信

### 3. CAN2 总线通信

| 功能    | 引脚     | 复用功能   | 方向 | 说明      |
| ------- | -------- | ---------- | ---- | --------- |
| CAN2_TX | **PB13** | AF9 (CAN2) | 输出 | CAN2 发送 |
| CAN2_RX | **PB12** | AF9 (CAN2) | 输入 | CAN2 接收 |

- **波特率**: 500 Kbps（与 CAN1 相同配置）
- **过滤器范围**: CAN2 使用 14~27 号（SlaveStartFilterBank=14, FilterBank=14）
- **其他参数**: 与 CAN1 完全一致（见 CAN1 配置）
- **用途**: 与热管理系统（TEMS）或其他扩展 CAN 设备通信

### 4. ADC1 模拟采样

| 通道                  | 引脚    | 模式     | 说明           |
| --------------------- | ------- | -------- | -------------- |
| ADC1_IN1 (电池总电流) | **PA1** | 模拟输入 | 采集电池总电流 |
| ADC1_IN2 (电池总电压) | **PA2** | 模拟输入 | 采集电池总电压 |
| ADC1_IN3 (预充电压)   | **PA3** | 模拟输入 | 采集预充电压   |

- **ADC 时钟**: PCLK2 / 4 = 84MHz / 4 = 21 MHz
- **分辨率**: 12位（0~4095）
- **采样时间**: 480 周期（高精度模式）
- **单通道转换时间**: (480 + 12) / 21MHz ≈ 23.4 μs
- **转换模式**: 单次转换，软件触发，逐通道独立配置
- **用途**: 配合 10ms 周期采样任务实时监控电池参数

### 5. LED 指示灯

| 功能         | 引脚    | 模式           | 说明             |
| ------------ | ------- | -------------- | ---------------- |

## 四、软件任务架构

### FreeRTOS 任务列表

| 任务名称        | 入口函数           | 优先级 | 堆栈(字) | 周期       | 功能说明                              |
| --------------- | ------------------ | ------ | -------- | ---------- | ------------------------------------- |
| **BMS_Main**    | `Task_BMS_Main`    | 5      | 256      | ~20ms 轮询 | BMS 五状态机主控任务                  |
| **ADC1_Sample** | `Task_ADC1_Sample` | 3      | 256      | 10ms 固定  | ADC1 三通道采集（电流/电压/预充电压） |

### 任务间通信机制

| 机制       | 句柄/标识符          | 方向                     | 用途                                           |
| ---------- | -------------------- | ------------------------ | ---------------------------------------------- |
| **事件组** | `g_bms_event_group`  | 外部任务 → BMS_Main      | 状态切换信号：充电/放电/故障/故障清除/预充完成 |
| **队列**   | `g_adc_sample_queue` | ADC1_Sample → 消费者任务 | ADC 采样数据（深度5, 类型 `ADC_SampleData_t`） |

### 事件组位定义

| 位名称                     | 掩码     | 设置方                 | 等待方        | 说明                     |
| -------------------------- | -------- | ---------------------- | ------------- | ------------------------ |
| `BMS_EVENT_CHARGE_REQ`     | `1 << 0` | CAN/IO 任务 (外部触发) | BMS_Main 任务 | 充电请求信号             |
| `BMS_EVENT_DISCHARGE_REQ`  | `1 << 1` | CAN/IO 任务 (外部触发) | BMS_Main 任务 | 放电请求信号             |
| `BMS_EVENT_FAULT`          | `1 << 2` | 保护/诊断任务          | BMS_Main 任务 | 故障信号（触发保护断开） |
| `BMS_EVENT_CLEAR_FAULT`    | `1 << 3` | 外部清除指令           | BMS_Main 任务 | 故障清除信号             |
| `BMS_EVENT_PRECHARGE_DONE` | `1 << 4` | 电压监测任务           | BMS_Main 任务 | 预充完成信号             |

### BMS 状态机

```
                        ┌──────────┐
            故障信号     │   IDLE   │◄────────── 故障清除
        ┌──────────────►│   空闲    │◄─── 请求撤销
        │               └────┬─────┘
        │                    │ 充电/放电请求
        │               ┌────▼──────────┐
        │   超时/故障    │  PRECHARGE    │
        │  ┌────────────│   预充电      │
        │  │            └────┬──────────┘
        │  │                 │ 预充完成
   ┌────▼──▼──┐       ┌──────▼──────┐
   │  FAULT   │       │ RUN/CHARGE  │
   │  故障    │       │  运行/充电   │
   └──────────┘       └─────────────┘
```

状态切换条件与动作：

| 当前状态  | 触发事件            | 目标状态      | 继电器动作                       |
| --------- | ------------------- | ------------- | -------------------------------- |
| IDLE      | 充电/放电请求       | PRECHARGE     | 闭合主负 + 预充，启动 3s 计时    |
| IDLE      | 故障信号            | FAULT         | 全部释放，吸合故障继电器         |
| PRECHARGE | 预充完成信号        | RUN 或 CHARGE | 断开预充 → 等待 500ms → 闭合主正 |
| PRECHARGE | 超时(3000ms) / 故障 | FAULT         | 全部释放                         |
| RUN       | 故障信号            | FAULT         | 全部释放，吸合故障继电器         |
| RUN       | 放电请求撤销        | IDLE          | 断开主正 + 主负                  |
| CHARGE    | 故障信号            | FAULT         | 全部释放，吸合故障继电器         |
| CHARGE    | 充电请求撤销        | IDLE          | 断开主正 + 主负                  |
| FAULT     | 故障清除信号        | IDLE          | 释放故障继电器，回到安全状态     |

**关键时序**：预充超时 3000ms | 预充完成保持 500ms | 状态机轮询 20ms

### ADC 采样数据流

```
Task_ADC1_Sample (精确 10ms 周期, vTaskDelayUntil)
    │
    ├──► adc1_read_bat_current()        → PA1 → 电池总电流
    ├──► adc1_read_bat_voltage()        → PA2 → 电池总电压
    ├──► adc1_read_precharge_voltage()  → PA3 → 预充电压
    │
    └──► xQueueSend(g_adc_sample_queue, &sample_data)
              │  (若队列满则丢弃最旧数据, 确保实时性)
              ▼
        消费者: BMS_Main 等任务通过 xQueueReceive() 消费
```
